## P0 — 项目初始化与基础设施搭建

> 🎯 目标：搭建 monorepo 工程结构，配置开发工具链

### 任务清单

- [ ] **Monorepo 初始化**
  - 使用 Turborepo 初始化 monorepo 结构
  - 配置 `turbo.json` 定义构建管道
  - 配置根 `package.json` 工作区

- [ ] **前端应用创建**
  - `npx create-next-app@latest apps/web` — Next.js 14 + App Router + TypeScript
  - 集成 Tailwind CSS
  - 集成 Shadcn/UI 组件库
  - 配置基础 `tsconfig.json`

- [ ] **后端应用创建**
  - 在 `apps/api` 下创建 Express/Hono 项目
  - 配置 TypeScript + ESLint
  - 配置 nodemon / tsx 热重载

- [ ] **共享包配置**
  - `packages/shared` — 共享类型定义
  - `packages/config` — 共享 ESLint / TypeScript / Prettier 配置

- [ ] **开发环境**
  - 编写 `docker-compose.yml`（PostgreSQL + Redis）
  - 配置 `.env.example` 环境变量模板
  - 配置 `.gitignore`

### 交付标准
- `npm run dev` 可同时启动前端 + 后端
- PostgreSQL + Redis 可通过 Docker 一键启动
- ESLint / Prettier 配置统一

## P1 — 前端核心页面开发

> 🎯 目标：完成用户端所有核心页面 UI

### 任务清单

- [ ] **全局布局**
  - Header / Navbar 组件（Logo、导航链接、搜索框、用户菜单）
  - Footer 组件（版权信息、社交链接、二维码）
  - 响应式布局适配（Mobile / Tablet / Desktop）

- [ ] **首页 `/`**
  - Hero Banner — 动态轮播 / 渐变背景
  - 技能分类 Tab 切换（全部、AI 绘画、AI 视频、AI 编程等）
  - 技能卡片网格 — 封面图 + 标题 + 描述 + 标签 + 价格
  - 搜索功能 — 关键词搜索 + 分类筛选
  - 分页 / 无限滚动加载

- [ ] **技能详情页 `/skill/[id]`**
  - 技能信息展示（标题、描述、价格、作者、标签）
  - 文件下载 / 预览区域
  - 相关技能推荐

- [ ] **社区页 `/community`**
  - 微信 / QQ 社群入口
  - 悬停显示二维码效果

- [ ] **关于页 `/about`**
  - 平台介绍 / 联系信息

### 交付标准
- 所有页面在 Chrome / Safari / Firefox 上正常展示
- 移动端自适应布局正常
- 页面使用 mock 数据渲染

## P2 — 后端 API 与数据库

> 🎯 目标：搭建数据库模型并实现核心 API

### 任务清单

- [ ] **数据库设计（Prisma Schema）**
  ```
  模型（Model）规划：
  ├── User        — 用户信息
  ├── Skill       — 技能资源
  ├── Category    — 技能分类
  ├── Tag         — 标签
  ├── BlogPost    — 博客文章
  ├── Comment     — 评论
  └── File        — 上传文件
  ```

- [ ] **Prisma 配置**
  - 初始化 Prisma，连接 PostgreSQL
  - 编写数据模型 Schema
  - 生成并执行数据库迁移
  - 编写 Seed 脚本（填充测试数据）

- [ ] **核心 API 开发**
  | API 路由 | 方法 | 说明 |
  |---------|------|------|
  | `/api/skills` | GET | 技能列表（分页、搜索、筛选） |
  | `/api/skills/:id` | GET | 技能详情 |
  | `/api/categories` | GET | 分类列表 |
  | `/api/tags` | GET | 标签列表 |
  | `/api/upload` | POST | 文件上传 |

- [ ] **tRPC 路由（可选）**
  - 配置 tRPC server + client
  - 定义类型安全的路由 procedures

- [ ] **中间件**
  - 错误处理中间件
  - 请求日志（Morgan / Pino）
  - CORS 配置
  - Rate limiting（Redis 限流）

### 交付标准
- 所有 API 可通过 Postman / REST Client 测试通过
- 数据库 migration 可重复执行
- Seed 数据可一键填充

## P3 — 博客系统

> 🎯 目标：完成完整的博客发布与阅读体验

### 任务清单

- [ ] **后端 — 博客 API**
  | API 路由 | 方法 | 说明 |
  |---------|------|------|
  | `/api/posts` | GET | 文章列表 |
  | `/api/posts/:slug` | GET | 文章详情 |
  | `/api/posts` | POST | 创建文章 |
  | `/api/posts/:slug` | PUT | 更新文章 |
  | `/api/posts/:slug` | DELETE | 删除文章 |
  | `/api/rss` | GET | RSS Feed |

- [ ] **前端 — 博客页面**
  - `/blog` — 文章列表页（分类筛选、搜索）
  - `/blog/[slug]` — 文章详情页（MDX 渲染、目录导航、代码高亮）
  - 文章卡片组件

- [ ] **MDX 支持**
  - 集成 `next-mdx-remote`
  - 自定义 MDX 组件（代码块、图片、提示框等）
  - 代码语法高亮（highlight.js / Shiki）

- [ ] **RSS 订阅**
  - 自动生成 RSS XML
  - 添加 RSS 链接到 Header

### 交付标准
- 博客文章支持 MDX 渲染
- RSS Feed 可被阅读器正常订阅
- 文章页支持 SEO（meta tags、OG tags）

## P4 — 用户认证系统

> 🎯 目标：实现完整的用户注册、登录与鉴权

### 任务清单

- [ ] **NextAuth.js 集成**
  - 配置 Auth Provider（GitHub / Google / 邮箱登录）
  - 配置 Session 策略（JWT）
  - 用户注册 / 登录 / 登出页面

- [ ] **后端 JWT 鉴权**
  - JWT Token 生成与验证
  - 受保护路由中间件
  - Refresh Token 机制

- [ ] **前端认证 UI**
  - 登录 / 注册弹窗或页面
  - 用户头像与下拉菜单
  - 已登录 / 未登录状态切换

- [ ] **权限管理**
  - 角色定义（普通用户 / 管理员）
  - 基于角色的路由守卫

### 交付标准
- 用户可通过 OAuth 或邮箱登录
- JWT Token 可正确签发与验证
- 受保护页面未登录时跳转登录

## P5 — CMS 管理后台

> 🎯 目标：搭建内容管理系统，支持技能和博客的后台管理

### 任务清单

- [ ] **Payload CMS 集成**
  - 安装配置 Payload CMS
  - 定义 Collections（Skills、Posts、Categories、Tags、Users）
  - 配置管理面板 UI

- [ ] **S3 文件上传**
  - 集成 AWS S3 / Cloudflare R2
  - 实现文件上传组件（拖拽上传、进度条）
  - 文件类型校验与大小限制

- [ ] **内容管理功能**
  - 技能资源管理（增删改查、上下架）
  - 博客文章管理（富文本编辑器、草稿 / 发布）
  - 分类与标签管理
  - 用户管理（查看 / 封禁）

### 交付标准
- 管理员可通过 CMS 面板管理所有内容
- 文件可成功上传至 S3 兼容存储
- 内容修改实时反映到前端

## P6 — 高级功能与优化

> 🎯 目标：提升用户体验和系统性能

### 任务清单

- [ ] **搜索优化**
  - 全文搜索（PostgreSQL pg_trgm / 外部搜索引擎）

- [ ] **缓存策略**
  - Redis 热门数据缓存
  - Next.js ISR 增量静态生成

- [ ] **SEO 优化**
  - meta tags / OG tags 动态生成
  - sitemap.xml 自动生成
  - robots.txt 配置
  - 结构化数据（JSON-LD）

- [ ] **性能优化**
  - 图片懒加载 + next/image 优化
  - 代码分割与动态导入
  - 首屏加载优化（LCP / FID / CLS）
  - 数据库查询优化 + 索引

- [ ] **用户体验增强**
  - 骨架屏（Skeleton）加载动画
  - 暗色模式 / 亮色模式切换
  - 页面过渡动画

### 交付标准
- Lighthouse 评分 ≥ 90（Performance / SEO / Accessibility）
- 首页加载时间 ≤ 2 秒

## 技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Payload CMS 与 Next.js 集成复杂 | 延期 1-2 天 | 可先用 markdown 文件系统替代 |
| tRPC 学习成本 | 延期 1 天 | 可先用 REST，后期迁移 |
| Redis 部署成本 | 增加基础设施成本 | 初期可用内存缓存替代 |
| S3 文件上传配置 | 延期 1 天 | Cloudflare R2 兼容 S3 API，成本更低 |

## 开发规范

### Git 工作流
- **主分支**：`main`（生产）/ `dev`（开发）
- **分支命名**：`feat/xxx`、`fix/xxx`、`chore/xxx`
- **提交规范**：Conventional Commits（`feat:` / `fix:` / `chore:`）

### 代码规范
- ESLint + Prettier 统一格式
- TypeScript strict mode
- 组件使用 PascalCase 命名
- API 路由使用 kebab-case 命名
